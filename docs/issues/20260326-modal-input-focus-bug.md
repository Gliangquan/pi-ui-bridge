# Issue: 页面有弹窗时，扩展面板输入框失效

## Goal

修复当页面显示弹窗（Modal）时，Pi UI Bridge 扩展面板中的输入框无法获得焦点和接收输入的问题。

## 背景/问题

当用户在浏览器页面中打开弹窗（如对话框、模态框等）时，Pi UI Bridge 扩展面板中的输入框变得无法交互。用户无法在输入框中输入文本或获得焦点，这严重影响了扩展面板的可用性。

## 验收标准 (Acceptance Criteria)

- [ ] WHEN 页面显示弹窗时，扩展面板输入框 SHALL 保持可交互状态
- [ ] WHERE 弹窗处于活跃状态，扩展面板输入框 SHALL 能够接收焦点
- [ ] IF 用户在扩展面板输入框中输入文本，THEN 文本 SHALL 正常显示

## 实施阶段

### Phase 1: 问题诊断
- [ ] 分析弹窗与扩展面板的 z-index 层级关系
- [ ] 检查事件冒泡和捕获是否被弹窗阻止
- [ ] 确认是否存在焦点陷阱（focus trap）问题

### Phase 2: 根本原因分析
- [ ] 查看 Pi UI Bridge 的 DOM 结构
- [ ] 检查 CSS 样式是否影响输入框的交互性
- [ ] 分析弹窗的事件处理逻辑

### Phase 3: 实施修复
- [ ] 调整 z-index 或 DOM 层级
- [ ] 修改事件处理逻辑
- [ ] 确保输入框在所有场景下都可交互

### Phase 4: 验证和测试
- [ ] 在不同类型的弹窗中测试
- [ ] 验证输入框的焦点管理
- [ ] 进行回归测试
